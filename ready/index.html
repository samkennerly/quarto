Build websites quickly.

A quarto is a

folder

containing:

proof
This is a proof of the finished website.
Open proof/index.html in a browser to see it.
quire
Source code is here.
[4 scripts + 1 module]
[quire folders can be swapped]
[see other forks: quartune, quaich, ...]
ready
Pages go here when they're ready to build.
Pages options go here too.
style
Stylesheets go here.
Use the default styles or roll your own.

Quarto is designed to:

  • build lean websites quickly
  • help beginners learn HTML and CSS
  • be modified, extended, and included in other projects
  • avoid installing 500 packages and a 500MB framework to build a 5kB page

Copy this quarto.

Quarto scripts require

Python 3.6+

. The clean script requires

Tidy 5+

.

with GitHub:
Login to GitHub and go to the quarto repo.
Click the big green Use this template button.
Create a new repo named badquarto.
Clone the badquarto repo to your local machine.
without GitHub:
Go to the quarto repo.
Click the big green Clone or download button.
Click Download ZIP, save the file, and unzip it to create a folder.
Rename the folder badquarto and drag it to wherever you want it.


You now have a copy of the stock quarto. There are many like it, but this one is yours.
To uninstall the stock quarto, delete its folder.
[open proof/index.html in a browser]
[delete Dockerfile, kitchen if you don't want Docker]

Edit your quarto.

Open a terminal and cd to wherever you put the Quarto folder.

Change the style.
quire/apply
[oops, that was all the styles]
quire/apply style/celestial
[or some other folder with CSS files]
Update the options.
Replace author with Wm. Shakespeare?.
Leave email blank. The Bard did not have email.
Options are optional. Don't want it? Leave it blank.
Any page can have an options file. See the examples folder for examples.
Rewrite a page.
Delete everything from index.html.
Add this tag: <img src="media/quarto.svg">
Delete the old pages.
Delete everything from the ready/ folder except index.html and index.json.
Add hamlet.html with this text: To be or not to be? Ay, there's the point.
Add hamlet.json with a title and description.
Update pages.txt, or delete it and use automatic pages.

Write a new quarto.

quire/delete
[can't delete empty folders because .DS_Store]
quire/clean
[caution: Tidy decapitates pages]
Tidy should warn you that, on page index.html, <img> is missing alt.
Add some alt text: <img alt="Quarto logo" src="media/quarto.svg">
[tidy again]
quire/build
quire/apply

Publish your quarto.

The proof folder is now a

static website

. To publish it, copy that folder to a

web host

. This site uses

neocities.org

, a non-profit which offers free and low-cost web hosting.

Q TIP: Working Directory Quarto scripts accept optional command-line arguments. By default, scripts assume they have been called from [ ]
Q TIP: Dynamic Websites Pages can be served dynamically by calling a Quarto Pages object from a Python micro-framework like bottle or flask. Quarto reads files iteratively and generate pages lazily, one line at a time. If these lines are streamed directly to browsers, then it is possible to serve large websites from cheap cloud instances with limited RAM and/or disk space.

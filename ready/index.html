Quarto builds websites.

Quarto is:
Quarto is designed to:
  • build lean websites quickly
  • help beginners learn HTML and CSS
  • be modified, extended, and included in other projects
  • avoid installing 500 packages and a 500MB framework to build a 5kB page
Quarto scripts require Python 3.6+. The clean script requires Tidy 5+.

Here's how to get one:
with GitHub:
Login to GitHub and go to the quarto repo.
Click the big green Use this template button.
Create a new repo named badquarto.
Clone the badquarto repo to your local machine.
without GitHub:
Go to the quarto repo.
Click the big green Clone or download button.
Click Download ZIP, save the file, and unzip it to create a folder.
Rename the folder badquarto and drag it to wherever you want it.
  • 1 static website
  • 4 CSS styles
  • 4 Python scripts
  • 1 Python module
Quarto's pages, styles, and all other files are contained in one folder. Quarto never modifies files outside its folder unless you tell it to. Multiple Quartos can exist on one machine without conflicts. To uninstall Quarto, delete its folder.

build a website

Open a terminal and cd to wherever you put the Quarto folder.

The target folder is now a static website. To publish it, copy that folder to a web hosting service. This site uses neocities.org, a non-profit which offers free and low-cost web hosting.
Q TIP: Dynamic Websites Pages can be served dynamically by calling a Quarto Pages object from a Python micro-framework like bottle or flask. Quarto reads files iteratively and generate pages lazily, one line at a time. If these lines are streamed directly to browsers, then it is possible to serve large websites from cheap cloud instances with limited RAM and/or disk space.

folders

preprint
This folder is for drafts, notes, outlines, etc.
Quarto never looks here unless you say so.
Delete this folder if you don't want it.
quarto
Source code is here. Edit these files to customize Quarto.
If you don't want to write Python code, that's OK! Don't edit these files.
To use someone else's customized Quarto, copy their quarto folder here.
ready
Write raw pages (HTML files) and page options (JSON files) here.
The clean script will tidy these files and extract the <body> of each page.
The build script uses each HTML file as the <main> element of a page.
style
Stylesheets (CSS files) go here. Use the default styles or roll your own.
The apply script will cat all CSS files in a folder into a single stylesheet.
Each page can use zero or more stylesheets, including external CSS.
target
Quarto writes to this folder. Open target/index.html to preview a site.
Upload this folder to a hosting service to publish your website.
The delete script deletes all HTML files in this folder.

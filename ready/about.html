Much Ado About Quartos

Quartos are designed to:

  • build lean websites
  • help people learn HTML and CSS
  • be modified by Python programmers
  • be copied, pasted, and shared with anyone.

See the home page to get started quickly.

Self-promotion is not so vile a sin.

Quartos build 5k pages with 500 lines of code, not 500M monoliths with 500 dependencies.
Brevity is the soul of git.
1 module
+ 4 scripts
+ 0 ten-ton widgets
= 1 quire folder
Run cleanly, as a simple script should do.
No hidden variables.
No files written outside its folder (unless you insist).
No plugins, templates, frameworks, loggers, shims, wrappers, or dinguses.
Scripts are in the quire folder, not the /usr/local/Cellar/python/3.7.0/Frameworks/Python.framework/Versions/3.7/lib/python3.7/quarto/quire/bin/scripts folder.
Write out thy lines with stateless lazy-lists.
Cat strings without schlemieling.
Generate HTML lazily one line at a time.
Build pages without copying images, audio, video, or any other files.
Do not start any web servers, file watchers, event listeners, or busy loops.
What can be happier than to despise all package installs?
No installation.
Extras, not requirements.
No combinatorial explosions of implicit dependencies.
Build multiple quartos on the same machine without conflicts.

Use as much of this code as please you.

Delete a section, add a search bar, or create a style. Customize your quarto as you like it.

Installing:
Don't. Create a new repo.
Uninstalling:
Delete it. Out, out, brief quarto! It's but a working folder.
Upgrading:
Copy and paste. Quire code's one page, and all the scripts are stored in the same folder.
Requesting features:
Create a new repo and DIY. One template repo makes the whole world kin.

Quartos are templates to be copied and modified, not frameworks to be installed and configured. (Whether that's good or bad is a matter of taste.)

Give the devs their due.
